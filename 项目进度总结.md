## 一、这个项目是干什么的？

**投递助手**（英文名：AI Job Decision Engine）是一个 AI 求职决策工具。

核心问题：求职者面对一个岗位时，常常不知道"我该不该投？"——不清楚自己和岗位的真实差距，也不知道具体该怎么改简历。

核心价值：把简历和岗位描述（JD）丢进去，**30 秒内**给出一个完整的决策闭环：
- ✅ **判断**：这个岗位值不值得投（0-100 分 + 建议：投 / 改了再投 / 放弃）
- 🔍 **理解**：哪里匹配、哪里有差距、风险点在哪
- 🎯 **行动**：具体要怎么改，每条建议精确到"问题 → 原因 → 操作"

**这不是聊天机器人，也不是简历生成器。** 它只做一件事：帮你快速决策要不要投这个岗位。

## 四、项目架构

```
投递助手/
├── src/
│   ├── App.tsx              # 主状态机（input → loading → result / history）
│   ├── types.ts             # 核心数据类型定义（AnalysisResult 等）
│   ├── translations.ts      # 中英文 UI 文案
│   ├── components/
│   │   ├── Header.tsx           # 顶部导航（语言切换、历史入口）
│   │   ├── InputSection.tsx     # JD + 简历双输入框，支持文件上传
│   │   ├── AnalysisLoading.tsx  # 分析中 loading 动画
│   │   ├── ResultDisplay.tsx    # 分析结果完整展示
│   │   └── HistoryDashboard.tsx # 历史记录面板
│   └── lib/
│       └── gemini.ts            # Gemini API 调用封装（走代理）
├── functions/api/           # Cloudflare Pages Function（API 代理）
├── vite.config.ts
└── 投递助手.md              # 完整 PRD 文档
```

**页面状态流转：**
```
输入页 (input)
    ↓ 点击 Analyze
加载页 (loading)
    ↓ AI 分析完成
结果页 (result)
    ↓ 返回
输入页 (input)

Header → 历史页 (history) ←→ 结果页 (result)
```

## 五、如何拓展

### 5.1 添加新的文件解析格式
在 `src/lib/gemini.ts` 的 `extractTextFromFile` 函数中增加对新 MIME 类型的处理分支（如 Excel、纯文本等）。

### 5.2 扩展分析维度
修改 `src/App.tsx` 中的 Prompt 模板，在 Schema 里增加新字段（如"公司文化匹配度"、"薪资合理性"），同时在 `src/types.ts` 中同步更新 `AnalysisResult` 接口，然后在 `ResultDisplay.tsx` 中新增对应展示模块即可。

### 5.3 接入云端数据存储
Firebase 已经引入依赖。将 `src/App.tsx` 中的 localStorage 读写替换为 Firestore 操作，可实现跨设备历史同步和用户账户体系。

### 5.4 添加多岗位对比
在 `HistoryDashboard.tsx` 中增加"勾选多条记录对比"的交互，复用现有的 `AnalysisResult` 数据结构，用雷达图展示多岗位各维度评分对比。

### 5.5 更换或升级 AI 模型
只需修改 `src/lib/gemini.ts` 中的 `model` 字段，或者替换整个 `callGeminiProxy` 函数以接入 OpenAI / Claude 等其他 API，Prompt 格式兼容。

### 5.6 国际化拓展
`src/translations.ts` 中已有中英文键值对结构，添加新语言只需新增一个语言对象，并在 `Header.tsx` 中增加切换选项。

## 六、未来发展方向

### 近期可做（MVP 优化）
- **结果导出**：将分析报告导出为 PDF 或 Word（jsPDF 已引入）
- **用户账号体系**：接入 Firebase Auth，支持跨设备同步历史记录
- **岗位收藏**：对历史记录打标签、备注，方便管理多个在投岗位
- **错误重试优化**：失败时自动保留输入内容，一键重试

### 中期方向
- **多岗位横向对比**：选出几个历史分析结果，生成对比报告，帮助用户优先排序
- **简历片段优化建议**：针对具体的差距，生成改写建议（仅改片段，不改整份简历）
- **岗位趋势感知**：分析多次投递记录，总结用户的能力短板规律
- **浏览器插件**：直接在招聘网站（Boss直聘、LinkedIn）上一键分析当前 JD

### 长期方向
- **投递追踪**：记录每个岗位的投递状态（已投 / 约面 / 挂），形成完整求职漏斗
- **能力成长地图**：根据历次分析的差距，给出系统性提升路径
- **团队/企业版**：HR 侧使用——批量筛选简历，找出最匹配岗位的候选人
